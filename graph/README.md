# Z-ai-platform Graph Data API

> **Audience:** External tools (especially `Z-ai-graph-viewer`) that need to
> display the true state of the Z-ai-platform ID graph.
> **Updated:** On every push to `main` (automated via CI).
> **Format:** JSON (deterministic — same input produces same output).

---

## Why `graph/` (not `docs/_graph/` or `standards/_graph/`)?

Three locations were considered:

| Location                | Why rejected                                                                                                          |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------- |
| `standards/_graph/`     | Inside `standards` git submodule; `git add` from main repo refuses submodule pathspecs (exit 128).                    |
| `docs/_graph/`          | Already contains visual artifacts (SVG, PNG, DOT) for documentation. Mixing API data with docs is semantically mixed. |
| **`graph/` (this dir)** | Clear semantics: `graph/` = machine-consumable exports, `docs/_graph/` = human-readable visuals. Clean separation.    |


## How to consume

### TypeScript / React (e.g. Z-ai-graph-viewer)

```typescript
const URL = "https://raw.githubusercontent.com/stsgs1980/Z-ai-platform/main/graph/id-graph.json";

interface GraphData {
  version: string;
  effective_date: string;
  exported_at: string;
  summary: {
    ids_extracted: number;
    related_edges: number;
    aligned_with_edges: number;
    hard_pass: number;
    hard_fail: number;
    warnings: number;
  };
  ids: Array<{
    id: string;
    prefix: string;
    domain: string;
    number: string;
    file: string;
  }>;
  related_edges: Array<{ from: string; to: string; file: string }>;
  aligned_with_edges: Array<{ left: string; right: string; file: string }>;
}

export async function fetchGraph(): Promise<GraphData> {
  const response = await fetch(URL);
  if (!response.ok) throw new Error(`HTTP ${response.status}`);
  return response.json();
}
```

### Recommended: periodic refresh every 5 minutes

```tsx
import { useEffect, useState, useCallback, useRef } from "react";

const REFRESH_INTERVAL_MS = 5 * 60 * 1000;
const STALE_THRESHOLD_MS = 15 * 60 * 1000;

export function useGraphWithAutoRefresh(url: string) {
  const [state, setState] = useState({
    data: null as GraphData | null,
    status: "idle" as "idle" | "loading" | "success" | "error" | "stale",
    lastUpdated: null as Date | null,
    lastError: null as string | null,
    nextRefreshIn: REFRESH_INTERVAL_MS / 1000,
  });
  const intervalRef = useRef<NodeJS.Timeout | null>(null);
  const countdownRef = useRef<NodeJS.Timeout | null>(null);

  const fetchGraph = useCallback(async () => {
    setState((s) => ({ ...s, status: "loading" }));
    try {
      const response = await fetch(url, { cache: "no-cache" });
      if (!response.ok) throw new Error(`HTTP ${response.status}`);
      const data = await response.json();
      setState({
        data,
        status: "success",
        lastUpdated: new Date(),
        lastError: null,
        nextRefreshIn: REFRESH_INTERVAL_MS / 1000,
      });
    } catch (err) {
      setState((s) => ({
        ...s,
        status: "error",
        lastError: err instanceof Error ? err.message : String(err),
      }));
    }
  }, [url]);

  useEffect(() => {
    fetchGraph();
    intervalRef.current = setInterval(fetchGraph, REFRESH_INTERVAL_MS);
    countdownRef.current = setInterval(() => {
      setState((s) => ({ ...s, nextRefreshIn: Math.max(0, s.nextRefreshIn - 1) }));
    }, 1000);
    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current);
      if (countdownRef.current) clearInterval(countdownRef.current);
    };
  }, [fetchGraph]);

  const isStale =
    state.lastUpdated && Date.now() - state.lastUpdated.getTime() > STALE_THRESHOLD_MS;

  return { ...state, isStale, refresh: fetchGraph };
}

// Usage:
const { data, status, lastUpdated, nextRefreshIn, isStale, refresh } = useGraphWithAutoRefresh(
  "https://raw.githubusercontent.com/stsgs1980/Z-ai-platform/main/graph/id-graph.json",
);
```

## Refresh strategy

| Frequency          | Method                                   | When to use                |
| ------------------ | ---------------------------------------- | -------------------------- |
| On every page load | Direct fetch + browser cache             | Live dashboard             |
| **Every 5 min**    | `setInterval` + countdown UI             | **Recommended default**    |
| Every 1 hour       | fetch with `Cache-Control: max-age=3600` | Slightly delayed dashboard |
| On demand          | Manual refresh button                    | Admin tools                |

**Recommended:** periodic refresh every 5 minutes. It balances freshness
(CI takes ~3 min from push to JSON ready) with server load.

### Refresh timeline

```
T+0:00   Developer pushes to Z-ai-platform main
T+0:30   GitHub Actions CI starts
T+1:00   verify-id-graph.js --json generates new data
T+1:30   CI commits graph/id-graph.json
T+2:00   raw.githubusercontent.com is updated
T+2:30   graph-viewer's setInterval fires → fetches new data
T+3:00   User sees updated graph in browser

Total delay: ~3 minutes from push to visible.
```

## Stability contract

The JSON shape is governed by **verify-id-graph.js v1.1.x**. Within v1.1.x:

- `summary.*` fields are stable
- `ids[]`, `related_edges[]`, `aligned_with_edges[]` arrays are stable
- New fields MAY be added (consumers should ignore unknown fields)
- Field renames or removals require a major version bump (v2.0.0)

Pin to a specific version:

```
GET .../id-graph.json?v=1.1.6
```

When the script version changes, the `version` field in the response will
indicate the new shape. Consumers should check this field.

## Source of truth

This JSON is generated by `node standards/scripts/verify-id-graph.js --json`
in the `verify-id-graph.yml` GitHub Actions workflow, then committed back to
the repo on every push to main.

**The data in this file IS the state of Z-ai-platform at the corresponding
commit.** There is no transformation, no caching layer, no manual editing.

If `Z-ai-graph-viewer` shows X, that X is what is actually in Z-ai-platform.
